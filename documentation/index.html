What is KV Storage?

Key-Value (KV) storage is a simple database model where data is stored as pairs of unique keys and their corresponding values. Think of it like a giant dictionary or hash map that's distributed across the internet.

üîë Key

A unique identifier (like a filename or ID) that you use to retrieve your data. Must be unique within the namespace.

user:123 session:abc-def-ghi cache:homepage-en config:api-settings

üíæ Value

The actual data associated with the key. Can be text, JSON, binary data, or any serializable content.

{ "name": "John Doe", "email": "john@example.com", "preferences": {...} }

Basic KV Operations

// PUT - Store data await KV.put("user:123", JSON.stringify({ name: "Ahmed", country: "SA", preferences: { theme: "dark" } })); // GET - Retrieve data const userData = await KV.get("user:123", { type: "json" }); // DELETE - Remove data await KV.delete("user:123"); // LIST - Find keys by prefix const userKeys = await KV.list({ prefix: "user:" });

Why Use KV Storage?

‚úÖ Advantages

  • Ultra-fast global reads (10-50ms)
  • Infinite scalability
  • No server management needed
  • Built-in redundancy and reliability
  • Simple API - easy to use
  • Cost-effective for read-heavy workloads
  • Perfect for caching

‚ùå Limitations

  • Eventually consistent (not immediate)
  • No complex queries or joins
  • Limited to 25MB per value
  • No ACID transactions
  • Write operations are slower
  • Not suitable for relational data

Common Use Cases

üöÄ Caching

Store frequently accessed data like API responses, computed results, or user sessions for lightning-fast retrieval.

// Cache API response for 1 hour const cacheKey = `api:users:${userId}`; let userData = await KV.get(cacheKey, { type: "json" }); if (!userData) { userData = await fetchUserFromAPI(userId); await KV.put(cacheKey, JSON.stringify(userData), { expirationTtl: 3600 }); } return userData;

üåç Configuration Management

Store application settings, feature flags, or environment-specific configurations that need global access.

// Feature flags await KV.put("feature:dark-mode", "enabled"); await KV.put("feature:new-ui", "beta"); // App configuration await KV.put("config:api-limits", JSON.stringify({ requestsPerMinute: 100, maxFileSize: "10MB" }));

üë§ User Sessions

Store user authentication tokens, session data, or temporary user preferences across your global application.

// Store user session await KV.put(`session:${sessionId}`, JSON.stringify({ userId: "123", loginTime: Date.now(), permissions: ["read", "write"] }), { expirationTtl: 86400 }); // 24 hours

üìä Analytics & Metrics

Store counters, analytics data, or metrics that need to be accessed quickly from anywhere in the world.

// Page view counter const views = await KV.get("stats:page-views") || "0"; await KV.put("stats:page-views", String(parseInt(views) + 1)); // User country statistics const countryKey = `stats:country:${userCountry}`; const count = await KV.get(countryKey) || "0"; await KV.put(countryKey, String(parseInt(count) + 1));

KV Storage for Your Broker Project

In your Cloudflare Worker broker project, KV storage would be perfect for:

üè¶ Broker Data Caching

Cache processed broker information by country to avoid database queries on every request.

// Cache broker data for each country const cacheKey = `brokers:${country}:v2`; let brokers = await KV.get(cacheKey, {type: "json"}); if (!brokers) { brokers = await getBrokersFromDB(country); await KV.put(cacheKey, JSON.stringify(brokers), { expirationTtl: 3600 // 1 hour }); }

üìä Performance Metrics

Store cache hit rates, response times, and usage statistics for monitoring.

// Track cache performance await KV.put(`metrics:${date}`, JSON.stringify({ hits: 1547, misses: 203, avgResponseTime: 145, countries: ["US", "SA", "AE"] }));

üöÄ Page Cache

Cache complete HTML responses for different countries and routes.

// Cache complete pages const pageKey = `page:${route}:${country}`; let html = await KV.get(pageKey); if (!html) { html = await generatePage(route, country); await KV.put(pageKey, html, { expirationTtl: 7200 // 2 hours }); }

‚öôÔ∏è Configuration

Store feature flags, API keys, and dynamic configuration without code deployments.

// Dynamic configuration const config = await KV.get("config:main", { type: "json" }); const featureEnabled = await KV.get("feature:new-ui") === "true";
